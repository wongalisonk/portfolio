Police Shootings

Front-End Web Development

Introduction

To demonstrate how technology can be used to bring awareness to political and social issues, I was assigned to create a map visualization of police shootings that occur in the United States. Data visualization was an important skill to learn because although data is important, the reality is that most people do not bother to analyze it. Using a visual tool to display the information not only allows for the data to be understood more easily, but is more impactful. For example, seeing and understanding that each dot on the map represents a person that got shot is much more jarring than looking at a long data set of text.

The interactive map was created using Leaflet.js which is "an open-source JavaScript library for mobile-friendly interactive maps."



The Data

This project was created when I was still learning about the basics of front-end web development. As a result, although I could have done some interesting things with the data set, I opted simply to display the locations of all the shootings and the basic information on click (location, description of events). For better differentiation between whether the victim was injured in the shooting or killed, I decided to change the color of the circle where white indicated that the victim was injured, and red indicated that the victim was killed.

The opacity of the circles was meant to highlight the concentration of where the shootings occurred. The harder it was to tell the differences between circles indicated a higher concentration of shootings. I thought it would be interesting if people were able to view and sort the data by race/ethnicity. I added a menu on the upper right for viewers to manipulate the data based on ethnicity.

The Table

I was required to create a table that complemented the data visualization above. The table I created was a simplistic one, displaying the victims based on gender and on whether the victim was hit or killed. This was calculated by aggregating the data. I chose to display the gender because there was no meaningful way I could have displaying it in the visualization since I had already colored the circles based on outcome (hit or killed).
