Guess! Spotify

API Web Application

Introduction

One of the projects I worked focused on a more specific type of skill in web application: using an API. For this particular assignment, our professor tasked us to use the Spotify API in any way we could think of. The first part of the project, for me at least, was researching and seeing what resources the API provided as well as what data I could access. After that it was time spent brainstorming in order to fulfill the "creative" aspect of the project.

For the final project, I leveraged AngularJS to help with user experience aspects of the project (more details below).

To play Guess! Spotify click here.


Looking for Inspiration

For inspiration for my project, I scoured the API reference pages to see if anything in particular gave me a creative spark. One of the biggest restraints of the API was the fact that it only gave access to 30 second segments of a song rather than the full length. Eventually, I stumbled across information on how to obtain the top 10 tracks of an artist. I began to play around with the idea of creating a guessing game where the player would have to guess the top tracks of the artist of their choice.

While this idea sounded simple enough when I decided upon it, it soon became apparent that there were many nuance components I had failed to take into consideration. First, I had to find a way to search the name of the artist that the player inputted. After that I needed to extract that specific artist's unique Spotify ID. This was so I could access the top ten tracks which is returned in JSON format. After a lot of trial and error (and frustration) with numerous tabs open on my web browser, I eventually figured out how to get the data I needed.

Pushing on

Next was the randomization of the songs. I wanted to randomize the order the songs were displayed because this provided better user experience for people who wanted to try the game again using the same artist. It would not be as fun if you could simply memorize the order of the songs. In short, the randomization offered better playability overall.

The centerpiece of the guessing game was a way to play the 30 second snippet of the song without the title showing. The data set retrieved helpfully provides a link to each song and it was a matter of simply displaying the link as a play button. At the end when the player has inputted all of their guesses, it was a matter of revealing the song titles after clicking the button. This was done using AngularJS (which I had learned earlier in the class).

The Finishing Touches

AngularJS was a very important framework that I used to help with the smaller, finer points of the project. This included when items (such as the Submit button) were displayed when a game was being played or when they were hidden (such as when the player first arrives at the website). Extra attention was also required for other use cases such as restarting the game or if an artist was not found. These were taken into consideration to provide the best user experience possible when playing.
